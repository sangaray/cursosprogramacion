# Programming Elements and Data Types

This document compiles the core definitions, elements, and examples explained in the fourth episode of the introduction to programming course, translated entirely into English.

## Attributes and Data

### Attribute

- **Definition:** A specific characteristic or feature that defines an object and distinguishes it from another. Attributes generally provide a permanent identifier or structure over time.
- **Examples from Video:** Name, Last Name, Age, and Career.

### Data

- **Definition:** The actual value assigned to an attribute. Unlike attributes, data values can change, vary, or be modified over time (e.g., the name attribute can hold values like Victor, Lucas, or Juan).
- **Examples from Video:** Victor, Cruz, 25, Software Engineering.

## Fundamental Data Types

The video highlights the most important data types used to build functionalities within a program:

### 1. Integer (int)

- **Definition:** Whole numbers without fractional or decimal components. They can be either positive or negative.
- **Examples:** `2`, `45`, `-8`, `1000`, `7`, `6`, `5000000`.

### 2. Float (Floating-point)

- **Definition:** Numbers that contain fractional parts or decimal points. They represent exact values and can be either positive or negative.
- **Examples:** `2.5`, `5.3`, `2.3`.

### 3. Character (char)

- **Definition:** A single unit of information that represents an individual letter, number, symbol, or punctuation mark.
- **Examples:** Letters (`A`, `S`), digits used as symbols, or signs (`?`).

### 4. String (Character Chain)

- **Definition:** A sequence or collection of multiple characters joined together to form words, text, or expressions.
- **Examples:** `"Victor"`, `"Cruz"`.

### 5. Boolean

- **Definition:** A logical data type that can only hold one of two possible values: True or False. It represents binary choices or logical states.
- **Examples:** `True`, `False`.
